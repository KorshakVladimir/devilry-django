.. _examiner_restfulsimplifieddeadline_details_delete:

=============================================================================
DELETE /examiner/restfulsimplifieddeadline/id
=============================================================================

Delete a Deadline.

The underlying data model where the item is deleted from is defined in :class:`devilry.apps.core.models.Deadline`.
The :ref:`simplified` that the server forwards this request to is
:meth:`devilry.apps.examiner.simplified.SimplifiedDeadline.delete`.


********
Request
********

Example
################

.. code-block:: javascript

    DELETE /examiner/restfulsimplifieddeadline/id



Required request parameters
###########################

id
--------------

The unique identifier of a Deadline. You will typically get
this id in response from a search.


**************
Response
**************


On success
##########

As long as the Deadline is deleted without an error, the
response is *HTTP 200* with a JSON object containing the id in the response body.


Example
-------

.. code-block:: javascript

    200 OK

    {
        id: 10
    }


On error
########

On errors, we respond with one of the :ref:`restful_api_error_statuscodes`

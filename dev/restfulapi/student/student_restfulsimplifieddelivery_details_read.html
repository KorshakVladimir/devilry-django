GET /student/restfulsimplifieddelivery/id¶

Retrieve a Delivery.

The underlying data model where the item is retrieved from is defined in devilry.apps.core.models.Delivery. The Simplified API that the server forwards this request to is devilry.apps.student.simplified.SimplifiedDelivery.read().

Request¶

Example¶

GET /student/restfulsimplifieddelivery/id

Another example, this one uses the optional result_fieldgroups parameter.

GET /student/restfulsimplifieddelivery/id

{
    result_fieldgroups: ["assignment_group_users", "assignment", "period", "delivered_by", "deadline", "assignment_group", "candidates", "subject"]
}

Required request parameters encoded in the URL¶

id¶

The unique identifier of a Delivery. You will typically get this id in response from a search.

Optional request parameters¶

Optional request parameters are encoded as a JSON object and sent as the request body as shown in the example above.

Response¶

On success¶

Example¶

200 OK

{
    id: 15,
    number: 20,
    time_of_delivery: "2010-02-22 22:32:10",
    deadline: 15,
    successful: true,
    delivery_type: 20,
    alias_delivery: 15
}

Success response details¶

As long as the Delivery is deleted without an error, the response is HTTP 200 with the requested data in the body. The result always contains the following fields:

On error¶

On errors, we respond with one of the HTTP Error status codes.
